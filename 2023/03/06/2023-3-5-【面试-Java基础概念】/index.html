面试-Java基础概念

发表于2023-03-06|更新于2023-03-06|面试
|字数总计:747|阅读时长:2分钟|阅读量:

java语言有哪些特点?

1.面向对象（封装，继承，多态）

2.跨平台性（由JVM实现跨平台性）

3.支持多线程(c++没有内置的多线程)

4.支持网络编程且很方便

5.编译与解释并存

一次编写，到处运行，曾经是Java的宣传口号

但目前虚拟化技术的已经非常成熟了，比如通过docker就很容易实现跨平台

JVM，JDK，JRE

JVM

是指java虚拟机，是运行java字节码的虚拟机，它针对不同的系统有不同的实现(Windows，Linux，macOS)，其目的是为了同一份字节码能够运行出相同的结果，是java实现跨平台的关键

而且JVM不止一种，它是一种规范，只要遵循了JVM规范，就可以开发

每个公司，组织和个人都可以开发自己的JVM，我们常用的是oracle公司出品的hotspot虚拟机

其他的还有IBM的OpenJ9

和GraalVM

JDK

是 Java Development Kit 缩写，它是功能齐全的 Java SDK。它拥有 JRE 所拥有的一切，还有编译器（javac）和工具（如 javadoc 和 jdb）。它能够创建和编译程序。

JRE

是 Java 运行时环境。它是运行已编译 Java 程序所需的所有内容的集合，包括 Java 虚拟机（JVM），Java 类库，java 命令和其他的一些基础构件。但是，它不能用于创建新程序。

什么是字节码，采用字节码的好处是什么?

在java中，可以被jvm理解的代码就叫做字节码，也就是.class文件

字节码并不面向任何操作系统，它是面向jvm的，因此java程序无序重新编译即可在不同的操作系统上运行

java程序从源代码到执行的过程

为什么说 Java 语言“编译与解释并存”？

计算机只能识别机器语言，而我们写的都是高级语言，因此在运行程序前需要将其转换为机器语言，在针对转换时机的不用

我们可以将高级编程语言按照程序的执行方式分为两种

  • 编译型 ：编译型语言会通过编译器将源代码一次性翻译成可被该平台执行的机器码。一般情况下，编译语言的执行速度比较快，开发效率比较低。
  • 解释型 ：解释型语言会通过解释器一句一句的将代码解释为机器代码后再执行。解释型语言的可移植较高，运行效率较低

java和c++的区别

  • Java 不提供指针来直接访问内存，程序内存更加安全
  • Java 的类是单继承的，C++ 支持多重继承；虽然 Java 的类不可以多继承，但是接口可以多继承。
  • Java 有自动内存管理垃圾回收机制(GC)，不需要程序员手动释放无用内存
